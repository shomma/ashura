# ASHURA 購入者向け操作メモ

ASHURAは、番組表から記事候補を見つけ、需要と競合を確認し、Geminiで記事タイトル案と記事構成案を作るための画面です。外部投稿先や運用サイトとの連携は行いません。

## 事前確認

- 設定画面で `Gemini API` と `番組表取得` がOKになっていることを確認します。
- 左メニューの `登録キーワード` で、番組表検索に使う語句を確認します。

## 1. 登録キーワードを整える

1. 左メニューの `登録キーワード` を開きます。
2. 番組表の中から拾いたい人物名、番組名、グループ名、話題語を追加します。
3. 使わない語句はOFFにします。

## 2. 番組表検索を実行する

1. 左メニューの `番組検索` を開きます。
2. 期間と放送種別を選びます。
3. 番組表取得ボタンを押し、取得結果の一覧を確認します。

## 3. 需要競合調査を確認する

- スコア、検索需要、競合件数、allintitle、強いドメイン数を見て、記事化する候補を選びます。
- 候補一覧はCSV出力できます。

## 4. 記事下書きを生成する

候補を選び、`このネタで記事タイトルと記事構成を生成` を押します。生成結果には、タイトル案、導入文案、構成案、Markdown本文、再利用用プロンプトが含まれます。

## 注意

- 実行履歴の保存は行いません。DB肥大化を避けるため、番組検索と記事下書き生成では履歴を蓄積しません。
- 外部投稿、サイト登録、外部投稿先ドメイン登録は行いません。
- APIキーやログイン情報はリポジトリに保存せず、運用環境の環境変数で管理します。
